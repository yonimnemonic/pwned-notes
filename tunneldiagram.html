SSH Local Port Forwarding

Cómo acceder a servicios internos que no están expuestos
HTB Funnel — Starting Point

El Problema

PostgreSQL (puerto 5432) solo escucha en 127.0.0.1 → solo acepta conexiones locales. Desde tu máquina atacante no puedes llegar. Además, psql no está instalado en el target.

La Solución: SSH Tunnel (Local Port Forwarding)

Creas una "tubería" con SSH que conecta tu puerto local 1234 con el puerto 5432 del target. Así puedes usar tu psql local como si estuvieras dentro de la máquina.

1

Creas el túnel desde tu máquina

Ejecutas ssh -L 1234:localhost:5432 christine@target. Esto abre el puerto 1234 en tu máquina y lo enlaza con el 5432 del target a través de SSH.

2

SSH reenvía el tráfico

Todo lo que envíes a localhost:1234 en tu máquina viaja cifrado por el túnel SSH hasta el puerto 5432 del target, donde está PostgreSQL escuchando.

3

Te conectas con psql local

En otra terminal: psql -U christine -h localhost -p 1234. Tu psql cree que habla con un PostgreSQL local, pero SSH lo lleva al target.

4

Consultas la base de datos

\c secrets → SELECT * FROM flag; → flag obtenida. PostgreSQL no sabe que la conexión viene de fuera — para él es una conexión local.

Analogía

🐧 → 🔒🕳️ → 🖥️ → 🗄️

Imagina que PostgreSQL está en una habitación cerrada con llave dentro de un edificio (el target). Tú no puedes entrar directamente a esa habitación desde la calle. Pero tienes las llaves de SSH para entrar al edificio. Con el túnel, SSH te abre una ventanilla en tu casa (puerto 1234) que conecta directamente con esa habitación cerrada (puerto 5432). Tú hablas por la ventanilla y llegas a la base de datos sin moverte de casa.
